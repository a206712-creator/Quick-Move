Laboratory Activities

Lab 1 – User Interface Design

This lab focused on building a static user interface using Jetpack Compose. I learned how to organize layouts using Column, Row, Box, Spacer, and basic UI components.

Reflection: Lab 1 introduced me to modern Android UI development with Jetpack Compose. I learned how different composables work together to create a structured layout. This lab established the foundation required for all subsequent labs and projects.

Lab 2 – User Interaction

This lab introduced interactive components such as TextField and Button. User input could now update the interface dynamically through state management.

Reflection: Through this lab, I understood the importance of state management in Compose applications. User interactions could now trigger UI updates instantly. This experience helped me understand reactive programming concepts used throughout Android development.

Lab 3 – Material Design

This lab emphasized modern Material Design principles, including typography, color schemes, themes, cards, and visual consistency.

Reflection: Lab 3 taught me how visual design contributes to user experience. By applying Material Design principles, I was able to create cleaner and more professional interfaces. This significantly improved the overall quality of my application.

Lab 4 – Navigation and ViewModel

This lab focused on implementing multi-screen navigation and managing application data using ViewModel. Navigation Compose was used to move between screens while preserving UI state.

Reflection: Lab 4 was a major milestone because it transformed a simple application into a multi-screen mobile app. I learned how Navigation Compose works and how ViewModel helps preserve data during configuration changes. This lab also introduced me to the MVVM architecture that is widely used in modern Android development.

Lab 5 – Room Database Integration

This lab introduced persistent local storage using Room Database. The application was upgraded with Entity, DAO, Repository, Database, and ViewModel components following MVVM architecture.

Reflection: Lab 5 helped me understand how real applications store data permanently. I learned how Room Database works together with DAO, Repository, and ViewModel. This lab completed my understanding of local persistence and allowed my application to save booking history even after restarting the app.

Reflection & Learning Journey

Throughout this semester, I experienced significant growth in mobile application development. Starting from simple static user interfaces in Lab 1, I gradually learned how to create interactive applications, apply Material Design principles, implement navigation, manage state using ViewModel, and integrate persistent storage using Room Database.


The progression from individual labs to complete projects allowed me to understand how different Android components work together. By developing the Quick Move application, I gained practical experience in designing real-world solutions rather than merely learning theoretical concepts.


Project 2 was particularly meaningful because it introduced advanced technologies such as Firebase Firestore, Weather APIs, and GPS sensors. These integrations helped me understand how modern mobile applications communicate with cloud services, external systems, and device hardware.


Overall, this course has strengthened my technical skills, problem-solving abilities, and confidence in Android development. I now have a much clearer understanding of how to design, develop, test, and deploy a complete mobile application using modern development practices.
